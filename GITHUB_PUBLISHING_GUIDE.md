# GitHub Publishing Guide

## Suggested Repository Name

`EIED-AI-Benchmark`

## Suggested Repository Description

A 12-benchmark healthcare AI evaluation suite testing whether evidence actually supports eligibility, closure, safety, quality, coding, longitudinal conclusions, guideline applicability, and other high-stakes decisions.

## Publishing Checklist

- Confirm all twelve benchmark folders are present.
- Confirm the package manifest reports 360 source documents and 48 batch archives.
- Confirm source documents contain only fictional information.
- Upload the complete folder structure, not only the ZIP.
- Confirm `README.md` renders on the repository landing page.
- Add the portfolio PDF to featured materials when available.
- Use release tag `v1.0` for the final Version 1.0 portfolio release.
- Do not add an open-source license unless you intentionally want to grant reuse rights.
